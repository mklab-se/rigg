# Rigg 0.20 — OpenAPI Contract Validation + AI Features Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deep validation of `x-rigg-api` links against their OpenAPI specs, and the Ailloy-gated AI features: diff/pull/push explanations, interactive conflict-resolution assistant, doctor remediation advice (spec §9, §10). NL scaffolding (`--describe`) already shipped in 0.18.

**Architecture:** Spec validation extends `commands/validate.rs` with an `openapi` module in rigg-core (parse spec, list paths, request/response `data` properties). AI features live in `commands/ai_assist.rs` and hook into diff (summary), push/pull (conflict merge proposal), doctor (advice) — all gated on `ailloy config_tui::is_ai_active("rigg")` and a global `--no-ai` flag.

## Global Constraints
- Same gates as 0.18/0.19. AI never runs in `--non-interactive` mode except read-only explanations; AI output is always confirmed by the user before being written/pushed.

### Task 1: OpenAPI spec model (rigg-core `openapi.rs`)
`pub struct ApiSpec { pub paths: Vec<String>, pub request_data_props: Vec<String>, pub response_data_props: Vec<String>, pub open_props: bool }` + `pub fn load(path: &Path) -> Result<ApiSpec, String>` parsing OpenAPI 3.x JSON: collect `paths` keys; walk the first POST operation's request/response schemas to `values[].data.properties` (following local `$ref`s); `open_props` true when `additionalProperties` != false. Unit tests against the scaffold spec + a custom-props spec.

### Task 2: Contract checks in validate
For each WebApiSkill with `x-rigg-api`: spec loads; skill `uri` path (path component) matches one of `spec.paths` (suffix match); when `!open_props`: every skill `inputs[].name` ∈ request props and every `outputs[].name` ∈ response props → problems otherwise. Fixture tests (matching + mismatching skill).

### Task 3: `--no-ai` global flag + diff explanations
Global `--no-ai` in cli.rs/GlobalContext. `commands/ai_assist.rs`: `pub fn ai_on(ctx) -> bool` (active && !no_ai); `pub async fn explain_diff(diffs) -> Result<String>` (summarize the change list in plain language, ≤200 words). diff prints the summary after the report when AI is on (text format only).

### Task 4: Conflict-resolution assistant
In push interactive conflict menu add `[a] AI merge proposal`: sends baseline-normalized local + remote JSON, asks for merged JSON only; parses; shows semantic diff vs both; `[k] keep proposal → push` / `[d] discard`. Same helper reused by pull's conflict prompt (writes local file on accept).

### Task 5: Doctor advice
After doctor failures with AI on: `explain_doctor(failures)` prints tailored remediation notes.

### Task 6: Gate + release 0.20.0
fmt/clippy/test, CHANGELOG, bump, merge main, tag.
